# Retro: Sprint 03 — Provider Contracts + `smallhost` Parser Skeleton

**Date:** 2026-05-23
**Scope:** TASK-03.1 through TASK-03.6 — `providers` interface and
sentinel taxonomy, factory registry, `smallhost` constructor +
properties bag, path helpers + validators, Devil parser fixtures
(`www add/list/restart`, `vhost list`, `ssl add/del`, `mysql add/del`),
and HostingProvider method skeletons over a narrow `Executor` seam.

## What Worked Well

- **Sentinel-first design.** Declaring the full
  `providers.ErrXxx` taxonomy in TASK-03.1 before writing any adapter
  code meant every parser and method had a canonical place to map to
  — no last-minute `errors.New("...")` calls leaked through.
- **Narrow Executor interface.** Pulling SSH out of the adapter
  surface as a one-method `Executor` interface kept the method tests
  pure (no `ssh.Pool`, no `sshmock`) while leaving the production
  wiring trivial: `NewSSHExecutor(pool, target)` is a 3-line
  delegation. Behavioral surface and transport surface are decoupled.
- **Tripwire-prefixed fixture passwords.** Using
  `REDACTED-NEVER-A-REAL-SECRET-aBcD1234EfGh5678` instead of a fake
  random password means any future secret scanner will flag it if it
  leaks. The fixture stays useful for parser tests while being
  trivially auditable.
- **Defensive parser corpus from day one.** The `www_add_malicious`
  fixture (ANSI escapes, NUL/BEL bytes, `$(rm -rf /)` inside the
  domain field) caught a real subtle issue in the strip pipeline:
  the initial `stripAndNormalize` did not reject `0x7f` (DEL) or
  bytes ≥ `0x80`. The fixture forced me to add those branches before
  shipping.

## What Didn't Work / Friction Points

- **`commit-msg` hook + Polish maintainer style clash.** The
  Conventional Commits regex demands lowercase first letter after
  the colon. Subjects like `feat(providers): TASK-03.1 — interface`
  were rejected. The fix is mechanical (drop the task id into the
  body / `Refs:` footer) but the hook diagnostic could spell it out
  with a corrected example next sprint.
- **Path helper signature drift between docs and interface.** Sprint
  plan called them `DeployPath` / `LogPath` (etc.); `HostingProvider`
  uses `GetDeployPath` / `GetLogPath`. Reconciled by using the
  interface names for interface methods and keeping `EnvPath` /
  `StoragePath` as additional helpers — but the inconsistency cost
  me a refactor mid-task. Worth normalizing the next provider doc
  before drafting Sprint 04.
- **Fixtures vs. live capture.** Every fixture in this sprint ships
  with `captured: inferred`. The parsers are defensive about ANSI,
  CRLF, NUL bytes, and size caps, but the textual shapes themselves
  are drafted from docs/providers/smallhost.md. The day a real
  `devil` output diverges, we'll need to rewrite — but the parser
  surface is small enough that the cost is bounded.

## Surprises / Unknowns Discovered

- `DatabaseKind` works better as a `type DatabaseKind = string`
  alias than a named type. The canonical interface signature accepts
  `dbType string`; with an alias, callers can still use the named
  constants without forcing a conversion. A non-alias type would
  have forced `string(kind)` casts at every wizard call site.
- `gocritic`'s `stringConcatSimplify` and `err113` chime in on
  patterns that read fine to a human (`strings.Join` for two
  elements, ad-hoc `fmt.Errorf("...")` without a wrapped sentinel).
  Treating both as design feedback rather than noise produced
  cleaner code overall.
- `time.Now` as a package-level `nowFn` seam beats threading a
  `Clock` through `Provider`. `CheckStatus` is the only place that
  measures wall time today, so the global is local enough to be
  safe; we will revisit if we add more time-sensitive methods.

## Open Questions To Resolve

- Should the `Executor` interface live in `providers` (so every
  adapter shares it) or stay in `providers/smallhost`? Current
  decision: keep it local. The shape is provider-specific (commands
  are typed as plain strings; `cpanel`/`directadmin` may need
  structured requests later). Promote to the shared package only
  if a second adapter actually needs the exact same surface.
- Should `RemoveSubdomain` accept the panel's stderr "not found"
  line as well as stdout? Today `combine(stdout, stderr)` handles
  it. We do not have a live fixture to confirm where the panel
  writes that line — left as TODO until live capture.
- Do we need a CI gate that fails when a fixture `.txt` exists
  without its companion `.fixture.md`? Likely yes (Sprint 07
  doctor / docs tooling work), but parking until then.
