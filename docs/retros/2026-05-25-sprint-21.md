## What worked well

- **Two transports, one Reader interface.** Defining `uapi.Reader` (4-method closed interface) up front meant the HTTPS client (`uapi.Client`), the SSH fallback (`uapi.SSHFallback`), and the future composite (`uapi.Composite`) became drop-in interchangeable. The compile-time assertions (`var _ uapi.Reader = (*Client)(nil)` × 3) caught a missing method during development — would have surfaced in unit tests anyway but the build break is *immediate* feedback, not a 3-second `go test` cycle.
- **Generic `decodeListResponse[T]` collapsed 3 nearly-identical decoders.** The first cut had `decodePassengerApps` / `decodeMysqlDatabases` / `decodeSSLKeys` as three independent ~30-line functions covering modern wrapper + top-level array + legacy map-keyed shapes. `dupl` lint rejected them; the refactor to a single generic accepting `wrapperKey` + `nameFromKey` (callback to promote legacy map keys into the row's `Name` field) eliminated ~60 lines and made adding a new module trivial. **Lint-driven design improvement** for the second sprint in a row (Sprint 20 had the same dynamic with `updateKey` cyclomatic complexity).
- **`shellQuote` defence-in-depth even though no untrusted input flows through it today.** Sprint 21 only constructs SSH commands from typed `Module` / `Function` constants, but the `'\''`-escaping `shellQuote` is wrapped around every shell argument anyway. The 10-case test table (`$(rm -rf /)` / `` `whoami` `` / `O'Brien` / `; ls; #` / `\backslash\`) documents the threat model for the next person who wonders "wait, is this safe?". Future contributors get the protection for free when they wire user-typed arg maps.
- **Composite generics removed runtime type assertions.** First cut of `tryComposite` used `any` returns + `.( *DomainInfoListResponse)` casts at every call site. Type-parameterising it to `tryComposite[T any](c *Composite, op func(Reader) (T, error)) (T, error)` collapsed each `List*` method to a one-line forward and gave the compiler the type info it needed to catch interface drift.
- **Smoke test caught a real bug.** `./bin/webox doctor cpanel --host=192.0.2.1 --user=test --token=FAKE --no-ssh --timeout=2s` was meant to be a "does the CLI compile and accept flags" probe. It revealed that 4 sections reported `FAILED` instead of `UNREACHABLE` because `transport.sleepWithCtx` was returning `ctx.Err()` unwrapped — `errors.Is(err, ErrTransportUnavailable)` returned false. The fix was a one-line wrap; the *discovery* needed a real end-to-end run because the unit tests injected zero-backoff and never exercised the context-cancel-during-retry path.

## What didn't work / friction points

- **`provider new cpanel <name>` vs `doctor cpanel` ambiguity took 2 attempts to resolve.** My first fix was reordering `parseArgs` to check the provider-name slot before `applySimpleFlag` — broke the `["provider", "new", "cpanel"]` case because `"new"` got captured as the provider name. The state-aware refactor (`simpleFlagHandled(parsed *opts, arg string)` + `isPendingProviderNameSlot(parsed)` helper) is the right shape but it required changing 5 call sites and writing 2 new helper functions. **Lesson: when a new contextual token would collide with an existing positional arg, the parser needs explicit state in the dispatch path; don't try to fix it with token reordering.**
- **`err113` lint forced 6 new typed sentinels for what were originally one-line `errors.New`s.** All six are now `var errCpanel*` package-level vars in `cpanel.go`. Pragmatically correct (operators can `errors.Is` against them in a future test harness) but feels heavyweight for messages that the CLI just prints to stderr. Either golangci-lint config needs an exception for `cmd/` packages or the codebase commits to typed sentinels universally — the current half-and-half is inconsistent.
- **`gocritic`'s `ifElseChain → switch` suggestion was correct but the suggested switch lost the original control-flow's two-branch symmetry.** Original was `if !noUAPI && token != "" { wire HTTPS }`; the switch version separates "noUAPI explicit" from "token absent" into two distinct cases that both feed the same `notes` slice. Slightly more code, same outcome — readability marginal in either direction.

## Surprises / unknowns discovered

- **cPanel UAPI legacy map shape is still in the wild.** Researched fixtures from public api.docs.cpanel.net document the modern `{"applications": [...]}` shape, but cPanel < 88 (still deployed on H88, Hosting88, some H88-resold accounts) returns `{"shop-backend": {...}, "marketing-cms": {...}}` keyed by row name. The decoder accepts both transparently with the `nameFromKey` callback. Won't know if this is actually exercised in production until TASK-21.7 onboards the live test account and we capture real fixtures.
- **`net/http`'s `httptest.NewTLSServer` + `srv.Client()` is enough to test full HTTPS path** without needing a real cert. The 24 unit tests in `client_test.go` run end-to-end TLS handshake + JSON envelope round-trip in ~5 seconds total. The Authorization-header check (no token leakage) is a defence test, not a transport test, and lives in the same suite.
- **`asciinema` cast non-deterministic across runs.** The recording script (TASK-21.6, shipped earlier in this session) produces a 13.6 KB cast each time, but the exact byte content differs because of timestamps and PTY scheduling. Acceptable for a demo artefact (no review value in a binary diff); not acceptable if we ever want to use casts in CI golden tests. If future sprints add cast-based regression checks, they'll need a `--deterministic-timing` flag pinned to a wallclock-independent timeline.
