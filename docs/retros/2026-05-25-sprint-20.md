# Retro: Sprint 20 — TUI Polish & Provider Catalog

**Date:** 2026-05-25
**Scope:** Closed all six Sprint 20 tasks (`TASK-20.1`..`TASK-20.6`) in a single autonomous session triggered by operator feedback that the TUI "feels unfinished" after Sprint 19. Plan: [docs/sprints/sprint-20-tui-polish-and-catalog.md](../sprints/sprint-20-tui-polish-and-catalog.md). Branch: `feat/sprint-20-tui-polish`.

## What worked well

- **Surface contract paid off twice.** `surface.Surface.Footer().Text` was a stable enough grammar (`[key] description · [key] description`) that the help overlay could parse it via a single 2-token regex without per-surface customization. Same contract drove the dashboard footer accuracy fix earlier in the sprint. One protocol, two features.
- **TDD on the help overlay caught a subtle modal-isolation bug.** The first cut of `handleOverlayKey` only intercepted `?`/`Esc`; pressing `n` or `tab` while the overlay was open would mutate underlying state (open new-project wizard, cycle tile focus). The "blocks underlying handlers" test made this visible immediately and forced the strict-block contract used in the host-key modal.
- **Refactor-then-add discipline.** When `updateKey` cyclomatic complexity hit 22 (lint gate at 20), extracting `handleOverlayKey` was the smallest possible change AND made the modal contract trivial to audit in isolation. Lint failure became a design improvement rather than a noise-suppression.
- **`cmd/screenshot` as stable dev tool.** Past sprints used ad-hoc `probe.go` scripts that got `git stash`-ed in/out; making it a long-lived `cmd/screenshot` binary means the next contributor regenerates baselines deterministically without having to remember the right `tea.KeyMsg` constructor incantation.
- **Clipboard avoided a new go.mod dep.** `os/exec` + per-OS allowlist (`pbcopy` / `xsel` / `xclip` / `wl-copy` / `clip.exe`) sidestepped the maintainer sign-off required for new modules per `AGENTS.md §5`. Charter constraint actually pushed to a leaner solution.

## What didn't work / friction points

- **Help overlay rendering attempt #1 was layered, not fullscreen.** Initial implementation appended overlay below `base` with `\n`. Looked correct in the model output but `View()`'s viewport renderer truncated to `screen.Height-1`, hiding the overlay entirely (regression caught only by inspecting the screenshot output, not by tests). Switched to fullscreen `lipgloss.Place` replacement. Lesson: **screenshot regen is a first-class test for any overlay change**, not a "doc nicety".
- **`gofumpt 0.10` × `go1.26` is brittle on multi-arg `append`.** Three call sites needed `slice = append(slice, helper...)` rewrites because the formatter rejected `append(slice, "literal", style.Render(…))` with mixed-arg style. Easy to fix but slows the loop; would have hit it anyway in CI. Could pre-empt with a pre-commit `gofumpt --check` in `.cursor/hooks/`.
- **Lint gate caught more than it should have on a docs/screenshot helper.** `cmd/screenshot/main.go` had four magic-number warnings (default width/height, exit codes) that don't really matter for a dev tool. Moved to named `const` block; cheap fix but feels like config noise. Could add an `mnd:exclude-cmd` rule to `golangci.yml` long-term — but only if it bites a third time.

## Surprises / unknowns discovered

- **`bento.Engine` differential rendering hides expected `teatest` needles.** The e2e test waiting for `[Active Projects]` after Esc'ing tile focus hung for the full 5s timeout because Bubble Tea's diff renderer optimized away re-emitting unchanging lines. Pivoted to `[Right/Enter] open` (footer-only, always re-emitted on focus exit). Documented now in `internal/e2e/cockpit_test.go`.
- **Help overlay surface label for dashboard is empty.** `dashboardSurface.Crumb()` returns `""` (intentional — bento engine paints its own "Webox Cockpit v0.1" header). Help overlay falls back to `"Cockpit"`. Fine for v0.1 but the operator-visible label is now duplicated logic across two files; if the dashboard ever gets a "real" crumb the help text will flip too.
- **`runtime.GOOS == "linux"` doesn't tell you Wayland vs X11.** Probing both `xsel`/`xclip` (X11) AND `wl-copy` (Wayland) was the only safe contract. Each fails fast on the wrong session, ~10ms penalty in the worst case.

## Changes to apply going forward

- **AGENTS.md §7.1 (gotchas):** add row "Bubble Tea differential rendering hides `teatest.WaitFor` needles when content is identical across frames — wait on a string the diff renderer is forced to re-emit (footer hint changes, status bar updates) rather than tile contents."
- **`.cursor/skills/tdd-loop`:** add note that for any TUI overlay change, regenerate `docs/screenshots/sprint-NN/*` and verify the overlay actually appears in the captured output before claiming the test suite is done. Tests that don't compare visual output can pass on overlay regressions.
- **Process:** screenshot regen is now a `make screenshot` target candidate (next sprint) — currently `go run ./cmd/screenshot --mode … --width … --height … > docs/screenshots/…/NN-…txt`. Consider adding a Make target with the canonical 14-screenshot recipe.
- **Tooling:** consider adding `gofumpt --check` to `.cursor/hooks/pre-commit.sh` so the `append` quirk surfaces locally instead of in `make lint`.
