## Co poszło dobrze

- **TDD na infra** się broni. `internal/version.Format` i
  `cmd/webox.Run` poszły red→green w <15 min każdy, table-driven testy
  wyłapały od razu „pusty version daje `dev` fallback” bez konieczności
  ręcznego smoke-testu binarki.
- **Izolowany `tools/go.mod`** rozwiązał rzeczywisty konflikt: `golangci-lint
  v2.12+` wymaga Go 1.26+, ale main module trzyma `go 1.24`. Dynamiczne
  `GOTOOLCHAIN` wyciągane przez `awk` z `tools/go.mod` w Makefile dało
  identyczny wynik na CI i lokalnie, bez `go install @latest`.
- **SHA-pinned actions + Dependabot z preserve-comment** robi tę pracę za
  nas. Każdy `uses:` ma 40-char SHA i `# vX.Y.Z` obok, więc bumpy w PR
  pokazują dokładnie którą wersję dostajemy.
- **`make ci` jako lokalny mirror CI**. Pojedynczy target obejmuje `tidy`,
  `lint`, `vet`, `vulncheck`, `test -race + cover-check`, `build`. Lokalnie
  zielony = remote zielony przy pierwszym push.
- **GoReleaser snapshot na bieżąco**. Cosign-placeholder (sh `echo` no-op)
  przeszedł `goreleaser check` v2 i wyprodukował poprawne `.sigstore.json`
  artefakty, więc kształt finalnego release jest już zwalidowany.

## Co poszło źle / było frustrujące

- **`make next-task` drift**. Skrypt patrzy na `[ ]` w plikach
  sprintów; AC zostały niezaznaczone aż do końca sprintu, więc skrypt
  uparcie zwracał `TASK-00.1` po faktycznym ukończeniu. Naprawione w
  ostatnim commicie tego sprintu (`docs(sprints): mark sprint-00 outcome`).
  **Lekcja:** AC zaznaczamy `[x]` w tym samym commicie co kod, nie post-mortem.
- **GoReleaser `signs.disable`**. Pierwsza próba dryrun-skip-signs przez
  `disable: '{{ .IsSnapshot }}'` wybuchła — pole nie istnieje w schemacie v2.
  Strata ~20 minut na walkę z YAML zanim zajrzałem do
  `goreleaser jsonschema`. **Lekcja:** zanim improwizuję z config field
  names, najpierw `goreleaser jsonschema | jq '.properties.signs'`.
- **macOS bash 3.2 vs `mapfile`**. Pre-commit hook posłałem z `mapfile`,
  bo zazwyczaj chodzi po bashu 5+ przez homebrew. Default macOS shell to
  bash 3.2 — `mapfile` nie istnieje, hook po prostu padał. Trzeba było
  fast-fixu (`fix(githooks)` jeszcze przed CI). **Lekcja:** każdy shell
  script targetujący `.git/` lub developer-machine musi przejść `bash 3.2`
  smoke-test (lub mieć `set -euo pipefail` + portable konstrukcje).
- **TASK-00.7 owner-rename ripple**. `github.com/webox/webox` →
  `github.com/dilitS/webox` bezpośrednio przed tagiem oznaczał trzy passy
  `Grep` + `replace_all` (kod, docs, skille, rules). Choć wszystko wlazło w
  jeden commit (`build(release): align module path with GitHub origin`),
  ryzykowałem rozjazdem między README a kodem. **Lekcja:** decyzja o
  module path to **TASK-00.0**, nie pre-tag clean-up. W każdym przyszłym
  sprincie 0 dla nowego modułu — owner picker pierwszy, `go mod init`
  drugi.

- **Sprint plan zakładał `tools.go` z blank imports.** Realnie
  `golangci-lint v2.12+` wymusił przejście na `tools/go.mod` (Go 1.24
  `tool` directive). Spec MVP `Tooling stack` (`AGENTS.md §1.1`) tego
  jeszcze nie odnotowywał — TASK-00.2 outcome dopisał decyzję, ale warto
  zaktualizować `AGENTS.md §1.1` przy najbliższej szansie.
- **`go vet` wymaga osobnego stepu w CI.** `golangci-lint v2` deklaruje
  `govet` jako enabled linter, ale pewne diagnostyki (np. `printf`)
  działają tylko gdy `go vet` jest uruchomione przeciw zbuildowanemu
  modulowi z resolved deps. Dorzucony osobny step w `lint` joba (`make vet`).
- **GitHub Actions `actions/cache@v5`** wymaga explicit `restore-keys:`
  z newline separator. Jeden `restore-keys: golangci-lint-${runner.os}-`
  wystarczył, ale dokumentacja sugerowała listę.
